## Project Overview

OpenGS Map Tool is a PyQt6 desktop application that generates procedural province and territory maps for grand strategy games. It uses Voronoi diagrams with Lloyd relaxation to create geographic subdivisions from input land/ocean mask images.

## Running the Application

```bash
pip install -r requirements.txt
python main.py
```

No test suite exists. Manual testing is done by running the app with example images in `example_input/`.

## Architecture

**Entry point:** `main.py` → creates `MainWindow` from `ui/main_window.py`

**Two-layer design:**
- `ui/` — PyQt6 GUI: 4-tab interface (Land Image, Boundary Image, Territory Image, Province Image) with sliders and buttons. State (images, metadata) lives as `MainWindow` instance attributes.
- `logic/` — Generation algorithms and I/O. No UI dependencies.

**Generation pipeline (territories first, then provinces):**
1. User imports land mask + boundary mask (PIL Images)
2. Territory generation: seed placement → Lloyd relaxation (4 iterations) → KDTree Voronoi assignment → border gap filling → color/metadata assignment. Stores `territory_pmap` (int32 index map) and `cached_masks` on the main window.
3. Province generation: distributes provinces proportionally across territories by pixel count, then subdivides each territory using the same Voronoi/Lloyd algorithm. Each province is tagged with its parent `territory_id`. Territory metadata is updated with `province_ids` lists.

**Shared algorithms** in `logic/utils.py`: `random_seeds()`, `lloyd_relaxation()`, `assign_regions()`, `extract_masks()`, `create_region_map()`, `combine_maps()`, `assign_borders()`, `color_from_id()`, `make_progress_updater()` — used by both generators.

**Key data structures:**
- Province metadata: list of dicts with `{province_id, province_type, R, G, B, x, y, territory_id}`
- Territory metadata: list of dicts with `{territory_id, territory_type, R, G, B, x, y, province_ids}`
- ID format: `PRV000001` / `TRT000001` (generated by `logic/numb_gen.py`)

**Export formats** (`logic/export_module.py`): PNG images, CSV definitions, JSON territory histories.

## Configuration

All tunable parameters are centralized in `config.py`: window sizing, slider ranges/defaults, algorithm iterations (`LLOYD_ITERATIONS`), color constants, image pixel limits, and ID prefixes. Always modify parameters here rather than hardcoding values in logic or UI code.

## Tech Stack

- **GUI:** PyQt6 (signals/slots, QApplication.processEvents for progress updates)
- **Computation:** numpy (arrays), scipy (KDTree, distance_transform_edt)
- **Imaging:** Pillow (PIL)
